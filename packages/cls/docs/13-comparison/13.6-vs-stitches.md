This chapter provides a **detailed comparison** between CLS and **Stitches**, a popular CSS-in-JS library with design token support. We'll explore their **similarities**, **differences**, and **when to choose each**.

## **What is Stitches?** ü§î

**Stitches** is a popular **CSS-in-JS** library that provides **styled components** with **design token support** and **variant-based styling**. It's designed to work seamlessly with **React** and provides excellent **TypeScript integration**.

### **Stitches' Core Philosophy** üéØ

- **CSS-in-JS approach** - Write styles in JavaScript with full CSS power
- **Design token system** - Native support for design tokens and themes
- **Variant-based styling** - Define variants and their combinations
- **TypeScript integration** - Full type safety for styled components
- **React-focused** - Built specifically for React applications

## **Side-by-Side Comparison** üìä

### **Basic Button Component** üéØ

Let's compare how both libraries handle a simple button component with size and tone variants.

#### **Stitches Implementation** üßµ

```typescript
import { styled } from "@stitches/react";

const StyledButton = styled("button", {
  // Base styles
  display: "inline-flex",
  alignItems: "center",
  justifyContent: "center",
  borderRadius: "0.375rem",
  fontWeight: "500",
  transition: "colors",
  outline: "none",
  cursor: "pointer",
  
  "&:focus": {
    boxShadow: "0 0 0 2px $colors$blue500, 0 0 0 4px $colors$white",
  },
  
  "&:disabled": {
    opacity: 0.5,
    cursor: "not-allowed",
  },

  // Variants
  variants: {
    size: {
      sm: {
        padding: "$2 $3",
        fontSize: "$sm",
      },
      md: {
        padding: "$2 $4",
        fontSize: "$base",
      },
      lg: {
        padding: "$3 $6",
        fontSize: "$lg",
      },
    },
    tone: {
      primary: {
        backgroundColor: "$blue600",
        color: "$white",
        "&:focus": {
          boxShadow: "0 0 0 2px $colors$blue500, 0 0 0 4px $colors$white",
        },
      },
      secondary: {
        backgroundColor: "$gray600",
        color: "$white",
        "&:focus": {
          boxShadow: "0 0 0 2px $colors$gray500, 0 0 0 4px $colors$white",
        },
      },
      danger: {
        backgroundColor: "$red600",
        color: "$white",
        "&:focus": {
          boxShadow: "0 0 0 2px $colors$red500, 0 0 0 4px $colors$white",
        },
      },
    },
  },
  
  defaultVariants: {
    size: "md",
    tone: "primary",
  },
});

interface ButtonProps extends React.ComponentProps<typeof StyledButton> {
  children: React.ReactNode;
}

function Button({ children, ...props }: ButtonProps) {
  return <StyledButton {...props}>{children}</StyledButton>;
}
```

#### **CLS Implementation** ‚ú®

```typescript
import { cls } from "@use-pico/cls";
import { useCls } from "@use-pico/cls/react";
import type { Component } from "@use-pico/cls";

// Define the CLS instance
const ButtonCls = cls({
  slot: ["root"],
  variant: {
    size: ["sm", "md", "lg"],
    tone: ["primary", "secondary", "danger"]
  }
}, ({ what, def }) => ({
  token: def.token({
    "color.bg.primary": ["bg-blue-600"],
    "color.bg.secondary": ["bg-gray-600"],
    "color.bg.danger": ["bg-red-600"],
    "color.text.primary": ["text-white"],
    "color.text.secondary": ["text-white"],
    "color.text.danger": ["text-white"],
    "spacing.sm": ["px-3 py-1.5"],
    "spacing.md": ["px-4 py-2"],
    "spacing.lg": ["px-6 py-3"]
  }),
  rules: [
    def.root({
      root: what.css([
        "inline-flex items-center justify-center rounded font-medium transition-colors",
        "outline-none cursor-pointer",
        "focus:ring-2 focus:ring-blue-500 focus:ring-offset-2",
        "disabled:opacity-50 disabled:cursor-not-allowed"
      ])
    }),
    def.rule({
      variant: { size: "sm" },
      root: what.css(["text-sm", "spacing.sm"])
    }),
    def.rule({
      variant: { size: "md" },
      root: what.css(["text-base", "spacing.md"])
    }),
    def.rule({
      variant: { size: "lg" },
      root: what.css(["text-lg", "spacing.lg"])
    }),
    def.rule({
      variant: { tone: "primary" },
      root: what.css(["color.bg.primary", "color.text.primary", "focus:ring-blue-500"])
    }),
    def.rule({
      variant: { tone: "secondary" },
      root: what.css(["color.bg.secondary", "color.text.secondary", "focus:ring-gray-500"])
    }),
    def.rule({
      variant: { tone: "danger" },
      root: what.css(["color.bg.danger", "color.text.danger", "focus:ring-red-500"])
    })
  ],
  defaults: def.defaults({
    size: "md",
    tone: "primary"
  })
}));

// React component
interface ButtonProps extends Component<typeof ButtonCls, React.ButtonHTMLAttributes<HTMLButtonElement>> {
  children: React.ReactNode;
  size?: "sm" | "md" | "lg";
  tone?: "primary" | "secondary" | "danger";
}

function Button({
  children,
  size = "md",
  tone = "primary",
  tva = ButtonCls,
  cls,
  ...props
}: ButtonProps) {
  const buttonClasses = useCls(tva, cls, ({ what }) => ({
    variant: what.variant({
      size,
      tone
    })
  }));

  return (
    <button className={buttonClasses.root()} {...props}>
      {children}
    </button>
  );
}
```

## **Key Differences Analysis** üîç

### **1. Architecture Philosophy** üèóÔ∏è

| Aspect | Stitches | CLS |
|--------|----------|-----|
| **Core Approach** | CSS-in-JS with styled components | Contract-first design with class-based styling |
| **Design Philosophy** | CSS-in-JS with design tokens | Type-safe and scalable class-based system |
| **Learning Curve** | Medium (CSS-in-JS concepts) | Medium (contract and token concepts) |
| **Scalability** | Good for CSS-in-JS workflows | Excellent for complex systems |

**Stitches**: **CSS-in-JS focused** - leverages the full power of CSS within JavaScript.

**CLS**: **Class-based focused** - provides type-safe styling with utility classes and design tokens.

### **2. Type Safety** üõ°Ô∏è

#### **Stitches Type Safety** üßµ

```typescript
// Stitches provides good type safety for styled components
const StyledButton = styled("button", {
  variants: {
    size: {
      sm: { padding: "$2 $3", fontSize: "$sm" },
      md: { padding: "$2 $4", fontSize: "$base" },
      lg: { padding: "$3 $6", fontSize: "$lg" }
    },
    tone: {
      primary: { backgroundColor: "$blue600", color: "$white" },
      secondary: { backgroundColor: "$gray600", color: "$white" }
    }
  }
});

// TypeScript infers component props
type ButtonProps = React.ComponentProps<typeof StyledButton>;
// Result: { size?: "sm" | "md" | "lg"; tone?: "primary" | "secondary"; ... }
```

#### **CLS Type Safety** ‚ú®

```typescript
// CLS provides comprehensive type safety
const ButtonCls = cls({
  slot: ["root"],
  variant: {
    size: ["sm", "md", "lg"],
    tone: ["primary", "secondary"]
  }
}, ({ what, def }) => ({
  // Contract validation ensures all variants are properly defined
  rules: [
    def.rule({
      variant: { size: "sm" }, // TypeScript validates this
      root: what.css(["text-sm"])
    })
  ]
}));

// Full type safety with contract validation
interface ButtonProps extends Component<typeof ButtonCls> {
  // TypeScript ensures all props match the contract
}
```

**Winner**: **CLS** - Provides **comprehensive type safety** with contract validation, while Stitches offers **good but limited** type safety.

### **8. Developer Experience** üë®‚Äçüíª

#### **Stitches Developer Experience** üßµ

**Pros:**
- **Full CSS power** - Leverage all CSS features in JavaScript
- **Design token support** - Native design token system
- **Good TypeScript support** - Type inference for styled components

**Cons:**
- **Learning curve** - More concepts to understand
- **Class-based approach** - May not appeal to CSS-in-JS fans
- **Complexity** - May be overkill for simple use cases
- **Utility class dependency** - Requires utility-first CSS approach

**Winner**: **Tie** - Depends on **preferred styling approach** and **project requirements**.

## **Use Case Recommendations** üéØ

### **Choose Stitches When You Need:** ‚úÖ

- **CSS-in-JS approach** with full CSS power
- **React-focused** styling solution
- **Design token system** with CSS-in-JS benefits
- **Dynamic styling** with JavaScript values
- **Scoped styles** and CSS-in-JS features
- **Projects where CSS-in-JS** is the preferred approach

### **Choose CLS When You Need:** ‚úÖ

- **Framework agnostic** styling solutions
- **Type-safe design systems** with complex inheritance
- **Runtime theming** and user customization
- **Multi-framework support** across different environments
- **Design token management** with inheritance and overrides
- **Complex conditional styling** with rule-based logic
- **Team collaboration** with predictable APIs
- **Large-scale applications** with complex styling requirements

## **Migration Complexity** üõ§Ô∏è

### **From Stitches to CLS** üìà

**Complexity**: üü° Medium (3-5 weeks)

**Key Challenges:**
- **Moving from CSS-in-JS to class-based** - Different styling paradigm
- **Learning contracts and tokens** - New concepts to understand
- **Architecture planning** - Designing inheritance hierarchy
- **Code restructuring** - Converting styled components to CLS contracts
- **Team training** - Teaching new patterns and concepts

**Migration Steps:**
1. **Install CLS** and remove Stitches dependency
2. **Create CLS contracts** for each styled component
3. **Convert CSS-in-JS styles** to utility classes
4. **Define design tokens** for consistent values
5. **Migrate component interfaces** to use `Component<T>`
6. **Replace styled components** with `useCls()` hook
7. **Test all variant combinations** work correctly

### **From CLS to Stitches** üìâ

**Complexity**: üü° Medium (3-5 weeks)

**Key Challenges:**
- **Moving from class-based to CSS-in-JS** - Different styling paradigm
- **Losing features** - No inheritance, limited runtime overrides
- **Code restructuring** - Converting CLS contracts to styled components
- **CSS-in-JS overhead** - Runtime CSS generation and injection
- **Performance impact** - Runtime CSS processing

```typescript
    "spacing.md": ["px-4 py-2"]
  }),
  rules: [
    def.rule({
      variant: { size: "sm" },
      root: what.css(["text-sm", "spacing.sm"])
    })
  ]
}));

// Works with React, Vue, Svelte, or any framework
function Button({ size, tva = ButtonCls, cls, ...props }) {
  const buttonClasses = useCls(tva, cls, ({ what }) => ({
    variant: what.variant({ size })
  }));
  return <button className={buttonClasses.root()} {...props} />;
}
```

## **The Bottom Line** üí°

### **Stitches is Perfect For:** üéØ

- **CSS-in-JS projects** with React
- **Dynamic styling** with JavaScript values
- **Design token system** with CSS-in-JS benefits
- **Scoped styles** and CSS-in-JS features
- **Teams preferring CSS-in-JS** approach
- **React-focused applications** with complex styling needs

### **CLS is Perfect For:** üéØ

- **Framework agnostic** styling solutions
- **Complex design systems** with inheritance and composition
- **Large-scale applications** requiring type safety and scalability
- **Teams building maintainable** and extensible styling systems
- **Projects requiring runtime theming** and user customization
- **Multi-framework environments** where consistency is crucial

### **The Choice Depends On:** ü§î

- **Styling approach** (CSS-in-JS vs class-based)
- **Framework requirements** (React-only vs multi-framework)
- **Project complexity** and scale
- **Team expertise** and learning capacity
- **Performance requirements** and constraints
- **Long-term maintainability** goals

**Stitches** is the **CSS-in-JS solution** for React applications with design tokens.

**CLS** is the **framework-agnostic solution** for complex design systems.

Choose the tool that matches your **styling approach** and **project requirements**! üöÄ

Token definition is where you **assign concrete values** to your token declarations. This chapter teaches you how to create meaningful token values, build token hierarchies, and establish design system foundations.

## üéØ What is Token Definition?

Token definition provides the **actual values** for your tokens. While declaration tells CLS *what* tokens exist, definition tells CLS *what values* those tokens have.

Think of it as **filling in the blanks** of your design vocabulary with actual colors, spacing, and other design values.

## üìù Basic Token Definition

```tsx
import { cls } from "@use-pico/cls";

const ButtonCls = cls({
  token: {
    // Nested token structure with groups and variants
    "color.bg": {
      primary: ["bg-blue-600"],
      secondary: ["bg-gray-600"],
      danger: ["bg-red-600"]
    },
    "color.text": {
      primary: ["text-white"],
      secondary: ["text-gray-900"],
      danger: ["text-white"]
    },
    "spacing": {
      sm: ["px-3 py-1.5"],
      md: ["px-4 py-2"],
      lg: ["px-6 py-3"]
    }
  }
}, ({ what, def }) => ({
  token: {},
  rules: [
    def.root({
      root: what.css([
        "inline-flex items-center justify-center rounded font-medium"
      ])
    })
  ]
}));
```

## üèóÔ∏è Token Value Patterns

### **Single Values**

Simple tokens with one CSS class:

```tsx
const SimpleTokens = cls({
  token: {
    "color.bg": {
      primary: ["bg-blue-600"]
    },
    "color.text": {
      primary: ["text-white"]
    },
    "spacing": {
      md: ["p-4"]
    },
    "border.radius": {
      md: ["rounded-md"]
    }
  }
});
```

### **Multiple Values**

Tokens that combine multiple CSS classes:

```tsx
const ComplexTokens = cls({
  token: {
    "color.bg": {
      primary: ["bg-blue-600", "hover:bg-blue-700"]
    },
    "color.text": {
      primary: ["text-white", "font-medium"]
    },
    "spacing": {
      md: ["px-4", "py-2"]
    },
    "border.radius": {
      md: ["rounded-md", "shadow-sm"]
    }
  }
});
```

### **Conditional Values**

Tokens that include responsive or state-based classes:

```tsx
const ResponsiveTokens = cls({
  token: {
    "spacing": {
      md: ["p-2", "md:p-4", "lg:p-6"]
    },
    "text.size": {
      md: ["text-sm", "md:text-base", "lg:text-lg"]
    },
    "border.radius": {
      md: ["rounded", "md:rounded-md", "lg:rounded-lg"]
    }
  }
});
```

### **Component-Specific Tokens**

Define tokens specific to components:

```tsx
const ButtonTokens = cls({
  token: {
    // Button sizes
    "button.size": {
      sm: ["px-3", "py-1.5", "text-sm"],
      md: ["px-4", "py-2", "text-base"],
      lg: ["px-6", "py-3", "text-lg"]

      primary: ["bg-blue-600", "text-white", "hover:bg-blue-700"],
      secondary: ["bg-gray-600", "text-white", "hover:bg-gray-700"],
      outline: ["border", "border-gray-300", "text-gray-700", "hover:bg-gray-50"],
      ghost: ["text-gray-700", "hover:bg-gray-100"]
    },
    
    // Button states
    "button.state": {
      disabled: ["opacity-50", "cursor-not-allowed"],
      loading: ["opacity-75", "cursor-wait"]
    }
  }
});
```

## üîç Best Practices

### **1. Consistent Naming**

```tsx
// ‚úÖ Good: Consistent naming pattern
const GoodTokens = cls({
  token: {
    "color.bg": {
      primary: ["bg-blue-600"],
      secondary: ["bg-gray-600"]
    },
    "color.text": {
      primary: ["text-blue-600"],
      secondary: ["text-gray-600"]
    }
  }
});

// ‚ùå Avoid: Inconsistent naming
const BadTokens = cls({
  token: {
    "bgPrimary": {
      default: ["bg-blue-600"]
    },
    "color.bg": {
      secondary: ["bg-gray-600"]
    },
    "textColor": {
      default: ["text-blue-600"]
    },
    "color.text": {
      secondary: ["text-gray-600"]
    }
  }
});
```

### **2. Semantic Values**

```tsx
// ‚úÖ Good: Semantic values
const SemanticTokens = cls({
```
